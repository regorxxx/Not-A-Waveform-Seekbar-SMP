UI panel which displays a seekbar, using a graphical representation of
the track via audiowaveform or ffprobe. May be based on RMS, peak levels,
actual waveform or visualization presets.

Mode:
-----
	• Audiowaveform: faster. Displays the waveform as is.

	• ffprobe: slower. Multiple analysis options available (RMS, peak
	 or RMS peaks).

	• Visualizer: may be used to show a random animation which changes
	according to BPM (if %BPM% tag exists). This mode does not analyze
	tracks.

Analysis data:
--------------
	• Analysis data is cached on files which are saved according to a
	configurable TF expression.

	• There are multiple settings to control how analysis data files
	are saved or auto-deleted.

	• Files are heavily compressed, using less than 2Kb in most cases.
	File size is greatly reduced in comparison to other waveform UI
	components storing similar data.

	• Since the script tries to load data files first, non compatible
	formats like SACD, ISO files or zipped files may be converted to
	compatible ones, analyzed and then deleted. As long as these
	incompatible tracks share the same path, the script will load the
	associated data files when playing them and display the seekbar
	properly (even if the track can not be analyzed on real time).

	• By default data is downmixed to mono, but there is also a
	setting to enable multi-channel analysis.

UI usage:
---------
	• Fully configurable using the R. Click menu.

	• Not recommended anything below 100 ms for refresh rate.

	• Variable refresh rate may be used to automatically adjust it
	according to system responsiveness.

	• There are multiple display options but all use the same data.

	• Independent waveform/panel background settings.

	• On multi-channel mode, channels shown may be selectively set.


Other UI usage:
---------------
	• Ctrl + Alt + Mouse Wheel to resize UI elements under mouse.

	• Shift + Win + R. Click to open SMP/JSplitter panel menu (works
	globally on any script and panel, at any position).

	• Ctrl + Win + R. Click to open script panel menu.

FAQ:
----
• VU Meter smoothness:
	◦ It's recommended to use ffprobe instead of Audiowaveform for
	the VU Meter preset, since its analysis data may have higher
	precision, resulting on a smoother animation.

• Some tracks are not analyzed:
	◦ Check 'Mode\Show Compatible extensions'. The list of compatible
	files depends on the mode (binary) used. You can convert them to
	a compatible format, analyze them and delete the converted tracks
	afterwards though.

	◦ Tracks within a compressed file are not supported; but if
	associated analysis files are found, they will be used. Like
	the example above, you can unzip and analyze them.

	◦ Check analysis files TF path at 'Analysis\File name format...'.
	Exotic paths, chars, artist names or setups should be handled by
	the users adjusting it to their needs.

• How to control dynamic colors artwork source:
	◦ It can be set at the background 'Art mode' sub-menu.

	◦ Selection/now playing source can be set at same sub-menu.

	◦ Alternatively colors may be provided by a third-party script
	if listen to color-servers setting is enabled.

• How to show the artwork as background instead of a solid color:
	◦ There are independent settings for panel/waveform background.

	◦ Background Art effect opacity must be non-zero.

	◦ Background Color mode must be set to none.

	◦ Both waveform background colors must be disabled (-none-).

	◦ Then background will be drawn based only on artwork type
	  selected. Then feel free to adjust the opacity, color mode
	  and effects.

• How to show make the background invisible (transparent panel):
	◦ There are independent settings for panel/waveform background.

	◦ Background Art effect opacity must be zero or Selection
	  mode set to None.

	◦ Background color mode must be set to none.

	◦ Both waveform background colors must be disabled (-none-).

	◦ Then background drawing will be skipped and panel may use
	  pseudotransparency if the JS host supports it. Note it must
	  be enabled at JSplitter/SMP configuration panel too.
